# リファクタリングチェックリスト

## 実行前チェック

### テストの確認
- [ ] 対象コードにテストが存在するか？
- [ ] テストがすべてグリーンか？
- [ ] カバレッジは十分か（80%以上推奨）？

### スコープの確認
- [ ] リファクタリング範囲が明確か？
- [ ] 一度に一つの改善に絞られているか？
- [ ] 依存関係を把握しているか？

### バックアップ
- [ ] Gitでコミット済みか？
- [ ] ロールバック可能な状態か？

## リファクタリング種別チェック

### Extract Method を適用すべきか？
- [ ] メソッドが30行を超えている
- [ ] コードブロックにコメントで説明が必要
- [ ] 同じコードが複数箇所に存在

### Decompose Conditional を適用すべきか？
- [ ] 条件式に&&/||が3つ以上
- [ ] ネストが3段階以上
- [ ] 条件の意図が読み取りにくい

### Introduce Parameter Object を適用すべきか？
- [ ] パラメータが4つ以上
- [ ] 同じパラメータ群が複数メソッドで使用
- [ ] パラメータに論理的関連がある

### Replace Temp with Query を適用すべきか？
- [ ] 一時変数が一度だけ代入される
- [ ] 計算ロジックが再利用可能
- [ ] 副作用がない

## 実行後チェック

### テストの確認
- [ ] すべてのテストがグリーンか？
- [ ] 新しいテストが必要か？

### 品質の確認
- [ ] コードの可読性が向上したか？
- [ ] 意図がより明確になったか？
- [ ] 重複が減少したか？

### メトリクスの確認
- [ ] 循環的複雑度が改善したか？
- [ ] メソッド行数が減少したか？
- [ ] パラメータ数が減少したか？

## 最終確認

- [ ] コードレビューの準備ができたか？
- [ ] 変更内容を説明できるか？
- [ ] ドキュメントの更新が必要か？
