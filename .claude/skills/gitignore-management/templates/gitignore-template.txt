# ═══════════════════════════════════════════════════
# Security Notes
# ═══════════════════════════════════════════════════

# このファイルをカスタマイズする際の注意事項:
#
# 1. 除外パターンの追加:
#    - 新しい機密ファイルタイプを発見したら即座に追加
#    - プロジェクト固有のSecretパスを追加
#
# 2. 例外パターン（!pattern）の慎重な使用:
#    - !.env.exampleのような例外は必要最小限に
#    - 例外パターンが実際に機密情報を含まないことを確認
#
# 3. ディレクトリパターン:
#    - secrets/（末尾スラッシュ）でディレクトリ全体を除外
#    - *secrets*で名前に含まれるすべてを除外（広範囲）
#
# 4. 検証:
#    - 追加後は必ずgit status で意図通り動作するか確認
#    - git add --dry-run .で追加されるファイルを事前確認
#
# 5. チーム共有:
#    - .gitignoreの変更は必ずチームに共有
#    - 全員が同じパターンを使用することを確認
