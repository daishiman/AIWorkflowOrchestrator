# Level 1: Basics

## 概要

Given-When-Then形式によるテスト可能な受け入れ基準の定義スキル。 要件の完了条件を明確化し、自動テストへの変換を可能にします。

SKILL.md の内容だけで完結する基本運用を扱います。

## 前提条件

- SKILL.md の概要とワークフローを読了している
- 対象タスクの目的と成果物を把握している

## 詳細ガイド

### 使用タイミング
- Use proactively when handling acceptance criteria writing tasks.

### 必要な知識
- 対象領域: Given-When-Then形式によるテスト可能な受け入れ基準の定義スキル。 要件の完了条件を明確化し、自動テストへの変換を可能にします。
- 主要概念: エッジケースパターン集 / 入力データのエッジケース / 1. 文字列 / Given-When-Then パターン集 / 基本構造
- 実務指針: 機能要件の完了条件を定義する時
- 実務指針: ユーザーストーリーに受け入れ基準を追加する時
- 実務指針: テストケースの基盤を作成する時

### 判断基準
- 避けるべき判断: アンチパターンや注意点を確認せずに進めることを避ける

### 成果物の最小要件
- テンプレートの必須項目を満たしている
- 主要テンプレート: `templates/acceptance-criteria-template.md`

### 参照書籍

### 主要リソース

### 主要テンプレート
