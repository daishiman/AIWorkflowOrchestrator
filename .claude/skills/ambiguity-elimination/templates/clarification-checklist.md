# 曖昧性明確化チェックリスト

## 概要

このチェックリストは、要件の曖昧性を体系的に明確化するための質問集です。
5つの曖昧性パターンごとに、ステークホルダーに確認すべき質問を提供します。

## 使い方

1. 曖昧な要件を特定
2. 該当するパターンのセクションを参照
3. 質問をステークホルダーに投げかける
4. 回答を基に要件を再記述

## チェックリスト完了後のステップ

### 1. 回答の整理

- [ ] すべての質問に対する回答を記録した
- [ ] 不明点や矛盾点を特定した
- [ ] 追加質問が必要な項目をリストアップした

### 2. 要件の再記述

- [ ] 量的曖昧性を数値と単位で明確化した
- [ ] 質的曖昧性を測定可能な基準で明確化した
- [ ] 範囲の曖昧性を完全列挙で明確化した
- [ ] 条件の曖昧性を条件式で明確化した

### 3. 検証

- [ ] 再記述した要件が測定可能である
- [ ] 再記述した要件がテスト可能である
- [ ] 再記述した要件にGiven-When-Then形式の受け入れ基準を作成できる
- [ ] ステークホルダーに確認し、合意を得た

## まとめ

このチェックリストを使用することで:

1. **体系的な質問**: 5つのパターンで網羅的に曖昧性を質問
2. **記録の一元化**: 質問と回答をドキュメントに記録
3. **検証可能性**: 明確化後の要件が測定・テスト可能

**ベストプラクティス**:
- 要件定義の早期段階でチェックリストを使用
- ステークホルダーとの対話で継続的に明確化
- 明確化レポートを要件ドキュメントに添付
