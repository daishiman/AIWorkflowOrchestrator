# 品質特性カタログ

## 概要

非機能要件は品質特性のカテゴリに分類されます。
このカタログは、FURPS+モデルとISO 25010を基に、
見落としがちな品質特性を網羅的にカバーします。

## FURPS+モデル

| カテゴリ | 説明 |
|---------|------|
| **F**unctionality | 機能性（FRとして分類） |
| **U**sability | 使用性 |
| **R**eliability | 信頼性 |
| **P**erformance | 性能 |
| **S**upportability | 保守性 |
| **+** | 設計制約、実装制約、インターフェース、物理制約 |

## 品質特性詳細

### 1. パフォーマンス（Performance）

**定義**: システムの応答速度とリソース効率

**サブカテゴリ**:

| 特性 | 説明 | 例 |
|-----|------|---|
| 応答時間 | リクエストから応答までの時間 | API応答200ms以内 |
| スループット | 単位時間あたりの処理量 | 1000リクエスト/秒 |
| レイテンシ | 処理の遅延時間 | ネットワーク遅延50ms以内 |
| リソース使用率 | CPU、メモリ、ディスクの使用率 | CPU使用率70%以下 |
| キャパシティ | 処理可能な最大負荷 | 同時接続10000ユーザー |

**質問例**:
- 許容される最大応答時間は？
- ピーク時の同時ユーザー数は？
- データ処理量の上限は？

**テンプレート**:
```markdown
## NFR: パフォーマンス

**応答時間**:
- API: 200ms以内（95パーセンタイル）
- 画面描画: 1秒以内（初期表示）

**スループット**:
- 通常時: 500リクエスト/秒
- ピーク時: 2000リクエスト/秒

**リソース**:
- CPU: 70%以下（通常時）
- メモリ: 80%以下
```

### 2. スケーラビリティ（Scalability）

**定義**: 負荷増加に対応する能力

**サブカテゴリ**:

| 特性 | 説明 | 例 |
|-----|------|---|
| 水平スケーリング | サーバー追加による拡張 | オートスケーリング対応 |
| 垂直スケーリング | リソース増強による拡張 | CPU/メモリ増設可能 |
| データスケーリング | データ量増加への対応 | 年間100TB対応 |
| ユーザースケーリング | ユーザー数増加への対応 | 100万ユーザーまで対応 |

**質問例**:
- 1年後、3年後の予想ユーザー数は？
- データ量の増加率は？
- ピーク時と通常時の差は？

### 3. セキュリティ（Security）

**定義**: データと機能の保護

**サブカテゴリ**:

| 特性 | 説明 | 例 |
|-----|------|---|
| 認証 | ユーザー識別 | OAuth 2.0、MFA |
| 認可 | アクセス制御 | RBAC、最小権限の原則 |
| 機密性 | データ保護 | AES-256暗号化 |
| 完全性 | データ改ざん防止 | ハッシュ検証 |
| 監査 | 操作記録 | 全操作のログ記録 |
| 否認防止 | 操作の証明 | デジタル署名 |

**質問例**:
- 認証方式の要件は？
- データ暗号化の要件は？
- 監査ログの保持期間は？
- コンプライアンス要件は？

**テンプレート**:
```markdown
## NFR: セキュリティ

**認証**:
- 方式: OAuth 2.0 + MFA（管理者必須）
- セッション: 30分タイムアウト、同時ログイン禁止

**暗号化**:
- 通信: TLS 1.3必須
- 保存: AES-256（個人情報）

**監査**:
- ログ保持: 5年間
- 対象: すべての管理操作
```

### 4. 可用性（Availability）

**定義**: システムの稼働状態の維持

**サブカテゴリ**:

| 特性 | 説明 | 例 |
|-----|------|---|
| 稼働率 | システムが利用可能な時間の割合 | 99.9%（年間8.76時間停止） |
| MTBF | 平均故障間隔 | 720時間 |
| MTTR | 平均復旧時間 | 1時間以内 |
| 冗長性 | バックアップシステム | アクティブ-スタンバイ構成 |
| フェイルオーバー | 障害時の切り替え | 30秒以内で切り替え |

**SLA対応表**:

| 稼働率 | 年間停止時間 | 月間停止時間 |
|-------|------------|------------|
| 99.0% | 87.6時間 | 7.3時間 |
| 99.9% | 8.76時間 | 43.8分 |
| 99.99% | 52.6分 | 4.38分 |
| 99.999% | 5.26分 | 26.3秒 |

**質問例**:
- 許容されるダウンタイムは？
- 計画停止の頻度と時間は？
- 障害時の目標復旧時間は？

**サブカテゴリ**:

| データ整合性 | データの一貫性維持 | トランザクション保証 |
| エラー処理 | 異常状態の適切な処理 | グレースフルデグラデーション |

**質問例**:
- 部分障害時の動作は？
- データ損失の許容範囲は？
- バックアップ頻度は？

### 6. 保守性（Maintainability）

**定義**: システムの変更・修正のしやすさ

| 特性 | 説明 | 例 |
|-----|------|---|

| 分析性 | 問題の特定しやすさ | ログ・トレーシング |
| 修正性 | 変更のしやすさ | デプロイ時間30分以内 |
| テスト容易性 | テストのしやすさ | 80%以上のコードカバレッジ |

**質問例**:
- コードカバレッジの目標は？
- ドキュメントの要件は？
- デプロイ頻度は？

### 7. 移植性（Portability）

**定義**: 異なる環境への移行能力

**サブカテゴリ**:

| 特性 | 説明 | 例 |
|-----|------|---|
| 適応性 | 異なる環境への対応 | マルチクラウド対応 |
| インストール容易性 | 導入のしやすさ | 自動インストーラー |

### 8. ユーザビリティ（Usability）

**定義**: 使いやすさと学習しやすさ

**サブカテゴリ**:

| 特性 | 説明 | 例 |
|-----|------|---|
| 学習容易性 | 使い方の習得しやすさ | チュートリアル完了率90% |
| 操作効率性 | 操作の効率 | 3クリック以内で目的達成 |
| エラー防止 | 誤操作の防止 | 確認ダイアログ |
| アクセシビリティ | 障害者への対応 | WCAG 2.1 AA準拠 |
| UI一貫性 | インターフェースの統一 | デザインシステム準拠 |

**質問例**:
- ターゲットユーザーの技術レベルは？
- アクセシビリティ要件は？
- 対応デバイスは？

### 9. 互換性（Compatibility）

**定義**: 他システムとの共存・連携能力

**サブカテゴリ**:

| 特性 | 説明 | 例 |
|-----|------|---|
| 相互運用性 | 他システムとの連携 | REST API提供 |
| 共存性 | 他システムとの共存 | 既存システムに影響なし |
| ブラウザ互換 | 複数ブラウザ対応 | Chrome, Firefox, Safari, Edge |
| デバイス互換 | 複数デバイス対応 | PC, タブレット, スマートフォン |

**質問例**:
- 連携する外部システムは？
- 対応ブラウザ・バージョンは？
- 対応OSは？
