# 階層設計ガイド

## 概要

階層設計は、概要から詳細へ段階的に深化する構造を設計する原則です。

## 設計原則

### 原則1: 概要から詳細へ

**構造**:
```
レベル1: 概要（SKILL.md）
└─ レベル2: 詳細（resources/）
   └─ レベル3: 高度な詳細（深いリソース）
```

### 原則2: 階層の深さ制限

**推奨**: 最大3階層

**理由**:
- ナビゲーションの複雑化を防ぐ
- 情報の発見可能性を維持

### 原則3: 各階層の独立性

**判断基準**:
- [ ] 各階層が独立して理解可能か？
- [ ] 上位階層を読まずに下位階層を理解できるか？

## 階層間の参照

**方法**:
- 上位から下位への参照: 明示的なリンク
- 下位から上位への参照: 「詳細はSKILL.md参照」

## ベストプラクティス

1. **階層は浅く**: 3階層以内
2. **各階層を独立させる**: 依存関係を最小化
3. **参照を明確に**: リンクを適切に配置
