# Level 1: Basics

## 概要

GitHub Actions ワークフローのセキュリティ強化スキル。

SKILL.md の内容だけで完結する基本運用を扱います。

## 前提条件

- SKILL.md の概要とワークフローを読了している
- 対象タスクの目的と成果物を把握している

## 詳細ガイド

### 使用タイミング
- Use proactively when handling workflow security tasks.

### 必要な知識
- 対象領域: GitHub Actions ワークフローのセキュリティ強化スキル。
- 主要概念: 権限強化（Permission Hardening） / デフォルト動作（リポジトリ設定依存） / 権限スコープ一覧 / サプライチェーンセキュリティ / サプライチェーン攻撃のリスク
- 実務指針: セキュリティ脆弱性の検出時（トークン露出、過剰な権限、未検証のアクション）
- 実務指針: ワークフローのセキュリティレビュー時
- 実務指針: PRワークフローの作成時（pull_request_targetの使用）

### 判断基準
- 避けるべき判断: アンチパターンや注意点を確認せずに進めることを避ける

### 成果物の最小要件
- テンプレートの必須項目を満たしている
- 主要テンプレート: `templates/secure-workflow.yaml`

### 参照書籍
- 『Web Application Security』（Andrew Hoffman）: 脅威モデリング

### 主要リソース
- `SKILL.md`: スキルの目的・前提・判断基準の基礎

### 主要テンプレート
