# コードレビューチェックリスト

## 命名

### 変数名

- [ ] 変数名は内容を明確に表しているか？
- [ ] 省略形や略語を避けているか？
- [ ] ブール変数はis/has/canで始まっているか？

### 関数名

- [ ] 関数名は動詞で始まっているか？
- [ ] 関数名が処理内容を正確に表しているか？
- [ ] 曖昧な名前（process, handle, do）を避けているか？

### クラス名

- [ ] クラス名は名詞か？
- [ ] クラスの責任を表しているか？

## 関数

### サイズ

- [ ] 関数は20行以下か？
- [ ] ネストは2段階以下か？
- [ ] パラメータは3つ以下か？

### 責任

- [ ] 関数は一つのことだけを行っているか？
- [ ] 抽象度が統一されているか？
- [ ] 副作用は最小限か？

## 重複

### コード重複

- [ ] 同じコードブロックが複数箇所にないか？
- [ ] 類似したロジックが繰り返されていないか？
- [ ] マジックナンバーがないか？

### DRY違反

- [ ] 同じ知識が複数箇所で表現されていないか？
- [ ] 変更時に複数箇所を修正する必要がないか？

## コメント

### 不要なコメント

- [ ] コードで表現できることをコメントしていないか？
- [ ] コメントアウトされたコードがないか？
- [ ] 古くなったコメントがないか？

### 必要なコメント

- [ ] 非自明な設計判断は説明されているか？
- [ ] 警告が必要な箇所にコメントがあるか？

## エラーハンドリング

- [ ] エラーケースは適切に処理されているか？
- [ ] エラーメッセージは明確か？
- [ ] 例外は適切な粒度でキャッチされているか？

## テスト

- [ ] テストは書かれているか？
- [ ] エッジケースはテストされているか？
- [ ] テスト名は何をテストするか明確か？

## 最終確認

- [ ] コードは読みやすいか？
- [ ] 意図は明確か？
- [ ] 保守しやすいか？
