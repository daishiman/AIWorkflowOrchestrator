# 要件完全性チェックリスト

## 概要

要件の完全性を確保するためのチェックリストです。
正常系だけでなく、異常系、境界値、エッジケースを網羅的にカバーします。

## シナリオ網羅性チェック

### 1. 正常系（Happy Path）
- [ ] 期待通りの入力での動作が定義されている
- [ ] 成功時の結果が明確に記述されている
- [ ] 成功時のシステム状態変化が定義されている

### 2. 異常系（Error Cases）
- [ ] 無効な入力での動作が定義されている
- [ ] エラーメッセージが具体的に定義されている
- [ ] エラー時のシステム状態が定義されている
- [ ] リカバリー方法が定義されている

### 3. 境界値（Boundary Values）
- [ ] 最小値での動作が定義されている
- [ ] 最大値での動作が定義されている
- [ ] 境界±1での動作が定義されている
- [ ] 空・NULL値での動作が定義されている

### 4. エッジケース
- [ ] 同時実行時の動作が定義されている
- [ ] タイムアウト時の動作が定義されている
- [ ] ネットワーク切断時の動作が定義されている
- [ ] 権限なしアクセス時の動作が定義されている

## 完全性スコアリング

### 評価基準
| カテゴリ | 重み | チェック項目数 |
|---------|------|---------------|
| 正常系 | 30% | 3項目 |
| 異常系 | 25% | 4項目 |
| 境界値 | 20% | 4項目 |
| エッジケース | 15% | 4項目 |
| 非機能要件 | 10% | 各カテゴリ |

### 目標
- 完全性スコア: >95%
- すべてのMust have要件: 100%
