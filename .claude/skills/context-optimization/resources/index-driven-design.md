# インデックス駆動設計

## 原則

SKILL.mdは「目次」として機能し、詳細はリソースに分離する。

## SKILL.mdの役割

### 役割1: 概要提供

何ができるかの全体像を提示

### 役割2: ナビゲーション

どこに何があるかを案内

### 役割3: クイックリファレンス

最も頻繁に使用する情報のみ含める

## 設計パターン

### パターンA: リソースマップ

````markdown
## リソース構造

\```
skill-name/
├── resources/
│ ├── basics.md # 基本的な使用法
│ ├── advanced.md # 高度な技法
│ └── troubleshooting.md # トラブルシューティング
\```

### リソース選択ガイド

**初めて使用**: basics.md から開始
**応用的な実装**: advanced.md を参照
**問題発生時**: troubleshooting.md を参照
````

### パターンB: チェックリスト駆動

```markdown
## クイックチェックリスト

- [ ] 正規化は適切か？ → 詳細: resources/normalization.md
- [ ] インデックスは最適か？ → 詳細: resources/indexing.md
```

## ベストプラクティス

1. **リソース構造セクション必須**: ディレクトリツリーを提示
2. **各Phaseで参照明示**: どのリソースを参照すべきか明確に
3. **選択ガイド提供**: ユーザーの判断を支援
