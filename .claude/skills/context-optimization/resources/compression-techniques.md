# 圧縮テクニック

## 原則

冗長性を排除し、本質的情報のみを抽出する。

## テクニック

### テクニック1: 重複の排除

**DRY原則の適用**:
```markdown
❌ 冗長:
Phase 1で詳細説明（200行）
リソースで同じ内容（200行）

✅ 圧縮:
Phase 1で概要のみ（30行）
詳細はリソースへ参照
```

**削減効果**: 60%

### テクニック2: 例の最小化

**原則**: 例は2-3個に限定

**構成**:
- 例1: 基本ケース
- 例2: 応用ケース
- (例3: エッジケース - 必要に応じて)

**削減効果**: 70%

### テクニック3: 概念要素中心

**原則**: 具体例より原則と判断基準

```markdown
❌ 具体的すぎる:
200行のサンプルコード

✅ 概念的:
原則: [原則名]
判断基準:
- [ ] [チェック項目1]
- [ ] [チェック項目2]
```

**削減効果**: 80%

## トークン見積もり

**計算式**:
```
トークン数 ≈ 行数 × 6トークン/行
```

## ベストプラクティス

1. **DRY徹底**: 同じ情報を書かない
2. **例は最小限**: 2-3個
3. **概念重視**: 原則と判断基準を中心に
