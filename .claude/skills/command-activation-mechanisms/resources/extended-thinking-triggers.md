# Extended Thinking トリガー

## 概要

Extended Thinking: モデルが深い推論を行うモード。コマンド内で特定のキーワードでトリガー可能。

## トリガーキーワード

### 明示的トリガー

```markdown
## Step 1: Analysis

Think carefully about the following:

- Architectural implications
- Performance trade-offs
- Security considerations
```

### その他の有効なフレーズ

- "Think carefully about..."
- "Consider thoroughly..."
- "Analyze deeply..."
- "Reason step by step..."
- "Think through..."

## 使用タイミング

✓ **適用すべき場面**:

- 複雑なアーキテクチャ決定
- セキュリティ分析
- パフォーマンス最適化
- トレードオフ評価

✗ **不要な場面**:

- 単純な実行タスク
- 定型的な操作
- 明確な手順がある作業

## 実装例

```markdown
## Step 2: Design Decision

Think carefully about the architecture:

- Which pattern best fits the requirements?
- What are the long-term maintenance implications?
- How does this scale?

Based on analysis, choose optimal approach.
```

## 注意点

- Extended Thinking はトークン使用量増加
- 本当に深い思考が必要な場合のみ使用
- 単純なタスクには不要
