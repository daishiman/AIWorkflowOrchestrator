# Level 2: Intermediate

## 概要

ベストプラクティスの収集、評価、統合、更新を体系的に行うスキル。

resources/・scripts/・templates/ の活用を前提とした運用を整理します。

## 前提条件

- Level 1 の内容を理解している
- SKILL.md の適用範囲を説明できる

## 詳細ガイド

### 必要な知識・情報
- 主要トピック: 情報源評価ガイド / 情報源の分類 / 一次情報源（最も信頼性が高い） / 統合戦略ガイド / 4つの統合パターン / パターン1: 追加型統合
- 実務指針: ベストプラクティスを収集して体系化する時 / 情報源の信頼性を評価する必要がある時 / 知識の品質を保証したい時

### 判断基準と検証観点
- 回避事項: アンチパターンや注意点を確認せずに進めることを避ける

### リソース運用
- `resources/information-source-evaluation.md`: 一次・二次・三次情報源の分類と信頼性評価基準、優先順位付けと採用判断の実践ガイド（把握する知識: 情報源評価ガイド / 情報源の分類 / 一次情報源（最も信頼性が高い））
- `resources/integration-strategies.md`: 4つの統合パターン（追加型・強化型・置換型・統合型）と重複排除戦略、知識統合のベストプラクティス（把握する知識: 統合戦略ガイド / 4つの統合パターン / パターン1: 追加型統合）
- `resources/quality-scoring.md`: 品質スコアリングガイド（把握する知識: 品質スコアリングガイド / 3軸評価 / 軸1: 正確性（40%の重み））
- `resources/legacy-skill.md`: 旧SKILL.mdの全文（把握する知識: Best Practices Curation / リソース構造 / リソース種別）

### スクリプト運用
- `scripts/log_usage.mjs`: 使用記録・自動評価スクリプト
- `scripts/validate-skill.mjs`: スキル構造検証スクリプト

### テンプレート運用
- `templates/evaluation-checklist.md`: ベストプラクティス評価チェックリスト

### 成果物要件
- テンプレートの構成・必須項目を反映する

## 実践手順

1. 利用するリソースを選定し、適用順を決める
2. スクリプトは `--help` で引数を確認し、検証系から実行する
3. テンプレートを使い成果物の形式を統一する
4. `scripts/log_usage.mjs` で実行記録を残す
