# 統合戦略ガイド

## 4つの統合パターン

### パターン1: 追加型統合

**定義**: 新規知識を独立した要素として追加

**適用条件**:
- [ ] 新規知識が既存知識と独立している
- [ ] トピックが異なる

**例**:
```
既存: A、B
新規: C
結果: A、B、C
```

### パターン2: 強化型統合

**定義**: 既存知識を補完・詳細化

**適用条件**:
- [ ] 新規知識が既存知識の詳細版
- [ ] 既存知識を強化する

**例**:
```
既存: Aの概要
新規: Aの詳細
結果: Aの概要 + 詳細
```

### パターン3: 置換型統合

**定義**: 古い情報を更新

**適用条件**:
- [ ] 新規知識が既存知識を更新
- [ ] 既存知識が陳腐化している

**例**:
```
既存: A（古い）
新規: A'（更新）
結果: A'
```

### パターン4: 統合型

**定義**: 複数の知識を統合する枠組みを提供

**適用条件**:
- [ ] 新規知識が既存知識を統合
- [ ] より高次の視点を提供

**例**:
```
既存: A、B
新規: C（AとBを統合）
結果: C（統合） + A参考 + B参考
```

## 重複の排除

**検出方法**:
1. キーワード検索
2. 概念的重複の確認
3. 情報源の重複チェック

**解消戦略**:
- マージ: 2つを1つに統合
- 参照: 主を定め、他は参照
- 削除: 劣る方を削除

## ベストプラクティス

1. **統合前に検索**: 既存知識を必ず確認
2. **重複排除**: DRY原則を徹底
3. **関連性明示**: 他の知識との関係を明確に
