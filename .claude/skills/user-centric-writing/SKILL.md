# User-Centric Writing スキル

## 概要

このスキルは、ユーザー中心のドキュメンテーション作成技術を提供します。
キャシー・シエラの思想に基づき、「機能を説明する」のではなく
「ユーザーを成功させる」ことを目的とした文書作成を支援します。

## 核心概念

### キャシー・シエラの5原則

1. **ユーザースキル向上優先**: 機能説明 < ユーザーができるようになること
2. **認知リソース管理**: ユーザーの脳のエネルギーを効率的に使う
3. **パフォーマンス指向**: 読んで理解するだけでなく、実際に行動できるまで
4. **信頼性構築**: ユーザーが自信を持って操作できるようにする
5. **漸進的開示**: 必要な時に必要な情報だけを提示

### タスク指向 vs 機能指向

| 機能指向（避ける） | タスク指向（推奨） |
|-----------------|-----------------|
| 「保存ボタン」 | 「作業内容を保存する方法」 |
| 「検索機能」 | 「必要な情報を見つける」 |
| 「設定画面」 | 「好みに合わせてカスタマイズする」 |

## ユーザーペルソナ分析

### 技術レベル分類

| レベル | 特徴 | 文書調整 |
|-------|------|---------|
| **初心者** | 専門用語不明、前提知識なし | すべて説明、画像多用 |
| **中級者** | 基本概念理解、応用に不安 | 概念は簡潔に、手順詳細 |
| **上級者** | 深い理解、効率重視 | リファレンス形式、例示 |

## 平易な言葉の原則

### 専門用語の扱い

```markdown
❌ 悪い例:
「APIエンドポイントにHTTP GETリクエストを送信してください」

✅ 良い例:
「データを取得するには、次のURLにアクセスします」
（補足: 技術的な用語が必要な場合は用語集へリンク）
```

### 能動態優先

```markdown
❌ 受動態:
「設定が保存されます」

✅ 能動態:
「システムが設定を保存します」
または
「保存ボタンをクリックして設定を保存してください」
```

### 文の長さ

- **目標**: 1文あたり25語以内
- **複雑な文の分割**: 複文は2-3の単文に分ける
- **段落の長さ**: 3-5文を目安

### 目的の明示

```markdown
❌ 何だけ:
「次へボタンをクリックします」

✅ 何と、なぜ:
「次へボタンをクリックして、入力内容を確認画面に進めます」
```

## 行動指向のライティング

### 明確な行動ステップ

```markdown
❌ 曖昧:
「適切な設定を行ってください」

✅ 具体的:
「1. 設定画面を開きます
2. 「通知」タブを選択します
3. 「メール通知を受け取る」をオンにします」
```

### 成功基準の明示

```markdown
✅ 良い例:
「この手順が完了すると、画面右上に「保存しました」と
表示されます。この表示を確認してから次に進んでください。」
```

## ドキュメント構造パターン

### タスク指向構成

```markdown
# [ユーザーの目標を達成する]

## はじめる前に
- 必要な準備や前提条件

## 手順
1. [具体的なアクション]
2. [具体的なアクション]
3. [具体的なアクション]

## 結果
- 成功時に何が起こるか

## うまくいかない場合
- トラブルシューティングへのリンク
```

### 認知負荷を減らす構造

1. **概要先行**: 全体像を最初に示す
2. **段階的詳細**: 必要に応じて深堀り
3. **視覚的区切り**: 見出し、箇条書き、空白を活用
4. **関連情報リンク**: 詳細は別ページへ誘導

## 品質チェックリスト

### 文章品質

- [ ] 専門用語を避け、一般的な言葉を使用しているか？
- [ ] 受動態ではなく能動態を使用しているか？
- [ ] 文は短く（1文25語以内推奨）、簡潔か？
- [ ] ユーザーの行動を明確に指示しているか？
- [ ] 「なぜ」その操作が必要かを説明しているか？
- [ ] 技術的背景ではなく、ユーザーの目的達成に焦点を当てているか？

### 構造品質

- [ ] タスク指向の見出しを使用しているか？
- [ ] 前提条件が明示されているか？
- [ ] 成功基準が記述されているか？
- [ ] 関連情報へのナビゲーションがあるか？

### ユーザビリティ

- [ ] 初心者が理解できる内容か？
- [ ] 5分以内で目標を達成できる構成か？
- [ ] 視覚的補助（画像、図）が適切に配置されているか？
