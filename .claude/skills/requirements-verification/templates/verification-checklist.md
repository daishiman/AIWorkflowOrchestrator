# 要件検証チェックリスト

## 一貫性検証

### データの整合性
- [ ] 同じデータの定義が一貫している
- [ ] データの単位が統一されている
- [ ] データの範囲が矛盾していない

### 用語の統一
- [ ] 同じ概念に同じ用語を使用
- [ ] 略語の定義が明確
- [ ] 用語集がある

### ビジネスルールの整合性
- [ ] 同じ条件で異なる結果を期待していない
- [ ] ビジネスルールが矛盾していない
- [ ] 優先順位が明確

## 完全性検証

### CRUD操作
- [ ] Create（作成）の要件がある
- [ ] Read（参照）の要件がある
- [ ] Update（更新）の要件がある
- [ ] Delete（削除）の要件がある

### エラー処理
- [ ] 入力エラーの処理が定義されている
- [ ] システムエラーの処理が定義されている
- [ ] ネットワークエラーの処理が定義されている
- [ ] 権限エラーの処理が定義されている

### 非機能要件
- [ ] パフォーマンス要件がある
- [ ] セキュリティ要件がある
- [ ] 可用性要件がある
- [ ] ユーザビリティ要件がある

## 検証可能性評価

### 測定可能性
- [ ] 曖昧な表現がない（目標90%）
- [ ] 具体的な数値と単位がある
- [ ] 測定方法が明示されている

### テスト可能性
- [ ] Given-When-Then形式の受け入れ基準がある
- [ ] 自動テストに変換可能
- [ ] 合否判定の基準が明確

## 追跡可能性確保

- [ ] すべての要件にIDがある
- [ ] 機能要件→ユースケースのリンクがある
- [ ] 機能要件→受け入れ基準のリンクがある
- [ ] 受け入れ基準→テストケースのリンクがある
