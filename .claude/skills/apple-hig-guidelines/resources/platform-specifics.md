# プラットフォーム別HIG対応

## プラットフォーム比較表

| 特性 | iOS | iPadOS | macOS | watchOS | visionOS |
|------|-----|--------|-------|---------|----------|
| 入力方式 | タッチ | タッチ/ポインター | マウス/トラックパッド | タッチ/Crown | 視線/ジェスチャー |
| ナビゲーション | タブバー | サイドバー | メニューバー | ページ | 空間配置 |
| 最小タップ | 44pt | 44pt | - | 44pt | **60pt** |
| 画面サイズ | 小〜中 | 中〜大 | 大 | 極小 | 無限 |
| マルチタスク | 限定的 | Split View | ウィンドウ | 単一 | 空間配置 |

---

## iOS

### 特徴
- モバイルファースト設計
- タッチ操作中心
- ポートレートモード重視
- Face ID/Touch ID対応

### ナビゲーション

**タブバー（推奨）**:
- 画面下部に配置
- 3〜5タブ
- 主要機能へのアクセス

**ナビゲーションスタック**:
- 階層的なコンテンツ構造
- 戻るボタンで履歴をたどる

### ジェスチャー

| ジェスチャー | アクション |
|------------|----------|
| 左端スワイプ | 戻る |
| 下スワイプ | 更新/閉じる |
| 長押し | コンテキストメニュー |
| ピンチ | ズーム |

### Face ID/Touch ID

**使用場面**:
- アプリロック解除
- 支払い認証
- センシティブデータアクセス

**設計ルール**:
- フォールバック（パスコード）を常に提供
- 認証理由を明確に説明

---

## iPadOS

### 特徴
- Split View/Slide Over対応
- ポインター対応
- サイドバーナビゲーション推奨
- Apple Pencil対応

### マルチタスク

**Split View**:
- 2アプリを並列表示
- サイズ比率: 50/50、70/30、30/70

**Slide Over**:
- アプリを重ねて表示
- 幅: iPhone相当

### サイドバー

**構造**:
```
[サイドバー] | [コンテンツリスト] | [詳細]
```

**設計ルール**:
- 第1列: ナビゲーション（常時表示）
- 第2列: リスト（折りたたみ可能）
- 第3列: 詳細コンテンツ

### ポインター対応

**ホバー効果**:
- ボタン: 微妙なハイライト
- リスト: 行のハイライト
- カーソル: 適切な形状変化

### iPadOS 26の新機能

**macOSライクなウィンドウ**:
- 信号機ボタン（赤/黄/緑）
- リサイズ可能なウィンドウ
- Stage Manager対応

---

## macOS

### 特徴
- メニューバー中心
- キーボードショートカット重視
- ウィンドウ管理
- Dock統合

### メニューバー

**標準メニュー**:
1. アプリケーションメニュー（アプリ名）
2. File
3. Edit
4. View
5. Window
6. Help

**設計ルール**:
- システム標準のメニュー項目を維持
- キーボードショートカットを提供
- コンテキストメニューとの整合性

### キーボードショートカット

**必須ショートカット**:

| ショートカット | アクション |
|--------------|----------|
| ⌘N | 新規 |
| ⌘O | 開く |
| ⌘S | 保存 |
| ⌘Z | 取り消し |
| ⌘C/V/X | コピー/ペースト/カット |
| ⌘Q | 終了 |
| ⌘W | ウィンドウを閉じる |
| ⌘, | 環境設定 |

### ウィンドウ

**ウィンドウコントロール（信号機ボタン）**:
- 🔴 赤: 閉じる
- 🟡 黄: 最小化
- 🟢 緑: フルスクリーン/ズーム

**最小サイズ**:
- 推奨: 400×300pt
- コンテンツに応じて設定

---

## watchOS

### 特徴
- 一目で把握できる情報設計
- Digital Crown操作
- コンプリケーション
- 小さい画面

### デザイン原則

**一目性**:
- 重要な情報を即座に表示
- スクロールを最小化
- 大きなタッチターゲット（44pt以上）

**シンプルさ**:
- 1画面1タスク
- 最小限のテキスト
- 高コントラスト

### Digital Crown

**用途**:
- スクロール
- ズーム
- 値の調整

### コンプリケーション

**種類**:
| サイズ | 用途 |
|-------|------|
| Circular | アイコン+数値 |
| Rectangular | 複数データ |
| Corner | 最小情報 |

### 画面サイズ

| モデル | サイズ |
|-------|-------|
| 41mm | 352×430px |
| 45mm | 396×484px |

## visionOS

### 特徴
- 空間コンピューティング
- 視線追跡
- ハンドジェスチャー
- 3D環境

### 最重要ルール

**タッチターゲット: 60×60pt以上**

visionOSでは視線追跡の精度を考慮し、
他のプラットフォームより大きなターゲットが必要。

### 入力方式

**視線追跡**:
- 要素を見つめてフォーカス
- ピンチでタップ

**ハンドジェスチャー**:
| ジェスチャー | アクション |
|------------|----------|
| ピンチ | タップ |
| ドラッグ | 移動 |
| 両手ピンチ+引き離し | ズーム |
| 手首回転 | スクロール |

### 空間設計

**水平優先**:
- 上下よりも左右を見る方が楽
- 水平方向のレイアウトを優先
- コンテンツは視線の高さに配置

**深度（Z軸）**:
- 近い: インタラクティブ要素
- 中間: メインコンテンツ
- 遠い: 背景

### ウィンドウ

**種類**:
| タイプ | 用途 |
|-------|------|
| Window | 標準的なアプリUI |
| Volume | 3Dコンテンツ |
| Full Space | 没入体験 |

**配置**:
- 適切な距離（1.5〜2m）
- ユーザーの視界中央

## クロスプラットフォーム設計

### 共通原則

1. **一貫したブランディング**
   - カラーパレットの統一
   - タイポグラフィの統一
   - アイコンスタイルの統一

2. **プラットフォーム適応**
   - 各プラットフォームの慣習を尊重
   - ナビゲーションパターンの適切な選択
   - 入力方式に最適化

3. **機能の優先順位**
   - 小さい画面では重要機能のみ
   - 大きい画面では追加機能を表示

### アダプティブレイアウト

**iPhone → iPad**:
- 1列 → 2〜3列
- タブバー → サイドバー
- モーダル → ポップオーバー

**iPad → Mac**:
- タッチ → マウス
- ジェスチャー → メニュー
- サイドバー → メニューバー統合
