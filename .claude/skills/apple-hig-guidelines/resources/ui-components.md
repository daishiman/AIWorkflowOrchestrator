# HIG UIコンポーネント仕様

## ボタンとタッチターゲット

### 最小タッチターゲットサイズ

| プラットフォーム | 最小サイズ  | 備考                 |
| ---------------- | ----------- | -------------------- |
| iOS              | 44×44pt     | 標準                 |
| iPadOS           | 44×44pt     | 標準                 |
| macOS            | 可変        | マウス操作前提       |
| watchOS          | 44pt        | 最小幅               |
| **visionOS**     | **60×60pt** | 視線追跡のため大きく |

### ボタン設計ルール

**タイトルの原則**:

- 動詞を使用（例: 「Save」「Cancel」「Delete」）
- タイトルケース（単語の先頭を大文字）
- 短く簡潔に（1〜3語）
- 曖昧な表現を避ける（「OK」より「Save Changes」）

**カラー規則**:

| 用途                 | カラー       | Hex     |
| -------------------- | ------------ | ------- |
| プライマリアクション | System Blue  | #007AFF |
| 破壊的アクション     | System Red   | #FF3B30 |
| キャンセル           | グレー系     | -       |
| 成功確認             | System Green | #4CD964 |

**ボタンスタイル**:

| スタイル | 用途                 |
| -------- | -------------------- |
| Filled   | プライマリアクション |
| Tinted   | セカンダリアクション |
| Bordered | 控えめなアクション   |
| Plain    | インラインアクション |

## ナビゲーションパターン

### タブバー（Tab Bar）

**配置**: 画面下部（iOS）

**推奨タブ数**: 3〜5個

**設計ルール**:

- 「Home」のような広すぎるラベルを避ける
- 具体的な名詞や動詞を使用
- タブバーとツールバーは同時に表示しない
- 選択状態を明確に表示

**ラベル例**:

- ✅ 「Today」「Library」「Search」「Settings」
- ❌ 「Home」「Main」「More」

**アイコンサイズ**:

- 標準: 25×25pt
- 大きめ: 31×31pt

---

### ナビゲーションバー（Navigation Bar）

**構造**:

```
[戻る] ← 左     [タイトル] ← 中央     [アクション] → 右
```

**高さ**:

| デバイス | スタイル       | 高さ |
| -------- | -------------- | ---- |
| iPhone   | 標準           | 44pt |
| iPhone   | ラージタイトル | 96pt |
| iPad     | 標準           | 50pt |

**戻るボタン**:

- システム標準の「<」シェブロンを使用
- 前画面のタイトルを表示（長い場合は「Back」）

**タイトル**:

- 中央配置
- 1行に収める
- ラージタイトルは左寄せ

**右アクションボタン**:

- 最大2個まで
- プライマリアクションを右端に

---

### サイドバー（Sidebar）- iPadOS/macOS

**用途**:

- マルチカラムナビゲーション
- コンテンツのカテゴリ分け

**設計ルール**:

- 第1列: ナビゲーション
- 第2列: リスト
- 第3列: 詳細

**幅**:

- 最小: 320pt
- 推奨: 320pt〜400pt

---

## モーダルとシート

### モーダル

**使用場面**:

- 集中が必要なタスク
- フローの中断が必要な場合
- 重要な確認

**設計ルール**:

- 使用は最小限に（ユーザーは非線形を好む）
- 完了ボタンとキャンセルボタンを常に含める
- タスクはシンプルかつ焦点を絞る
- 背景を暗くして焦点を示す

### シート（Sheet）

**iOS/iPadOS のシートスタイル**:

| スタイル       | 用途                 |
| -------------- | -------------------- |
| フルスクリーン | 複雑なフロー         |
| ページシート   | 標準的なモーダル     |
| フォームシート | フォーム入力（iPad） |
| カード         | 軽量なアクション     |

**Detent（部分表示）**:

- **Medium**: 画面の約半分（推奨）
- **Large**: フルスクリーン近く
- **Custom**: カスタム高さ

**チェックリスト**:

- [ ] シートの高さは適切か？
- [ ] スワイプでの閉じる操作が可能か？
- [ ] 完了/キャンセルボタンがあるか？

## アラートとアクションシート

### アラート（Alert）

**使用場面**:

- 重要で予期しない情報
- 即座の注意が必要な場合
- 確認が必要な場合

**設計ルール**:

| 要素           | 推奨                   |
| -------------- | ---------------------- |
| タイトル       | 短く（1行以内）        |
| メッセージ     | 必要に応じて（簡潔に） |
| ボタン数       | 1〜2個                 |
| プライマリ配置 | 右側                   |

**ボタン配置**:

- 2ボタン: キャンセル（左）、アクション（右）
- 3ボタン: 垂直スタック

**注意**: 乱用するとUXが低下する

---

### アクションシート（Action Sheet）

**使用場面**:

- 意図的なアクションに関連する選択肢
- 複数のオプションから選択

**表示形式**:

| デバイス | 表示位置       |
| -------- | -------------- |
| iPhone   | 画面下部       |
| iPad     | ポップオーバー |

**設計ルール**:

- キャンセルボタンを常に含める
- 破壊的アクションは**赤色で上部**に配置
- ボタン数は最大4個（キャンセル含む）

**ボタン順序**:

1. 破壊的アクション（赤、上部）
2. その他のアクション
3. キャンセル（下部、分離）

## リスト（List）

### スタイル

| スタイル      | 用途             |
| ------------- | ---------------- |
| Inset Grouped | 設定画面         |
| Plain         | シンプルなリスト |
| Sidebar       | ナビゲーション   |

### セル高さ

| タイプ           | 高さ     |
| ---------------- | -------- |
| 標準             | 44pt     |
| サブタイトル付き | 60pt     |
| 画像付き         | 80pt以上 |

### アクセサリ

| アクセサリ              | 用途       |
| ----------------------- | ---------- |
| ディスクロージャー（>） | 詳細画面へ |
| チェックマーク          | 選択状態   |
| 詳細ボタン（i）         | 追加情報   |
| スイッチ                | オン/オフ  |

## フォーム

### テキストフィールド

**設計ルール**:

- プレースホルダーで入力例を示す
- ラベルは上部または左側
- エラー状態を明確に表示
- キーボードタイプを適切に設定

**キーボードタイプ**:

| タイプ     | 用途           |
| ---------- | -------------- |
| Default    | 一般テキスト   |
| Email      | メールアドレス |
| Number Pad | 数字のみ       |
| Phone Pad  | 電話番号       |
| URL        | URLアドレス    |

### ピッカー

| タイプ            | 用途             |
| ----------------- | ---------------- |
| Date Picker       | 日付選択         |
| Wheel Picker      | 選択肢から選ぶ   |
| Segmented Control | 少数の排他的選択 |

---

## 検索

### 検索バー

**配置**: ナビゲーションバー下部

**機能**:

- スクロールで隠れる
- タップで展開
- キャンセルボタン表示

### 検索結果

- インクリメンタルサーチを推奨
- 検索候補を表示
- 最近の検索を保存

## プログレスインジケーター

### Activity Indicator（スピナー）

**用途**: 不確定な待機時間

**スタイル**:

- Medium: 20×20pt
- Large: 36×36pt

### Progress Bar

**用途**: 確定した進捗

**設計ルール**:

- 進捗率を視覚的に表示
- パーセンテージ表示はオプション

### Pull to Refresh

**用途**: コンテンツの更新

**設計ルール**:

- システム標準の動作を使用
- 更新中はスピナー表示
