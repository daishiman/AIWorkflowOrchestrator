# HIG レイアウト＆グリッドシステム

## レイアウト原則

### コンテンツ優先設計

1. **コンテンツがUI**
   - UIはコンテンツを支え、邪魔しない
   - 余白を活用して読みやすさを確保
   - 情報密度のバランス

2. **視覚的階層**
   - 重要な情報は大きく、目立つ位置に
   - 関連情報はグルーピング
   - 空白で区切りを表現

3. **一貫性**
   - グリッドに沿った配置
   - 反復パターンの使用
   - 予測可能なレイアウト

## グリッドシステム

### 基本グリッド: 8ptグリッド

すべての要素は8ptの倍数で配置:
- 4pt: 微細な調整のみ
- 8pt: 基本単位
- 16pt: 標準間隔
- 24pt: セクション間
- 32pt: 大きな区切り

### レスポンシブカラム

| 幅 | カラム数 | ガター | マージン |
|-----|--------|-------|---------|
| < 375pt | 4 | 8pt | 16pt |
| 375-428pt | 4-6 | 16pt | 16pt |
| 428-768pt | 6-8 | 16pt | 20pt |
| 768-1024pt | 8-12 | 20pt | 24pt |
| > 1024pt | 12-16 | 24pt | 40pt |

### コンテンツ最大幅

| コンテンツタイプ | 最大幅 | 理由 |
|----------------|-------|------|
| **本文テキスト** | 680pt | 可読性（1行65-75文字） |
| **カード** | 400pt | 視認性 |
| **フォーム** | 500pt | 入力しやすさ |
| **画像ギャラリー** | 1200pt | 表示領域確保 |
| **ダッシュボード** | 制限なし | 情報量優先 |

│  ┌─────────────────────┐    │

## アスペクト比

### 画像＆メディア

| タイプ | アスペクト比 | 用途 |
|-------|------------|------|
| **正方形** | 1:1 | プロフィール、サムネイル |
| **横長（標準）** | 16:9 | ビデオ、バナー |
| **横長（ワイド）** | 21:9 | シネマティック |

## レスポンシブブレークポイント

### Size Classes（iOS/iPadOS）

| Size Class | 幅 | 高さ | デバイス例 |
|-----------|-----|-----|----------|
| **Compact Width** | < 600pt | - | iPhone（縦） |
| **Regular Width** | ≥ 600pt | - | iPad、iPhone（横） |
| **Compact Height** | - | < 400pt | iPhone（横） |
| **Regular Height** | - | ≥ 400pt | iPhone（縦）、iPad |

## 整列＆配置

### テキスト整列

| 言語方向 | デフォルト | 数値 | ボタン |
|---------|----------|------|--------|
| **LTR** | 左揃え | 右揃え | 中央 |
| **RTL** | 右揃え | 左揃え | 中央 |

### アイコン配置

| コンテキスト | 配置 |
|------------|------|
| **リーディング** | テキストの前（LTRでは左） |
| **トレーリング** | テキストの後（LTRでは右） |
| **中央** | ボタン、ナビゲーション |
| **オーバーレイ** | 画像上のアクション |

### 垂直整列

| タイプ | 整列 |
|-------|------|
| **リスト項目** | 中央（テキストとアイコン） |
| **ボタン内容** | 中央 |
| **カードタイトル** | 上揃え |
| **フォームラベル** | ベースライン |
