# HIGの3つのテーマと6つの設計原則

## HIGの基本理念

### 「ヒューマンインターフェース」という名称の意味

1987年のHIGには「**人々はコンピュータを使おうとしているのではなく、仕事を完了しようとしている**」
という根本原則が明記されている。

Apple WWDC 2017での説明:
> 「userという言葉は臨床的で匿名化する効果を持ち、人々をインターフェースとの関係でのみ狭く定義する。
> humanはより豊かなニュアンスを想起させる」

---

## 3つのテーマ（iOS 7以降）

### 1. Clarity（明瞭性）

**定義**: あらゆるサイズで読みやすいテキスト、正確で理解しやすいアイコン、控えめで適切な装飾

**実践ポイント**:
- 強い視覚的階層がユーザーを重要な要素へ導く
- 曖昧さや混乱を排除する
- テキストは読みやすさを最優先
- アイコンは意味が明確でなければならない

**チェックリスト**:
- [ ] テキストは最小サイズでも読みやすいか？
- [ ] アイコンの意味は一目で理解できるか？
- [ ] 装飾は機能を妨げていないか？
- [ ] 視覚的階層は明確か？

---

### 2. Deference（謙譲性）

**定義**: UIがユーザーのコンテンツを主役にするために一歩引く姿勢

**実践ポイント**:
- 流動的なアニメーションと半透明のUI要素
- ボーダーや重いボタン、影は最小限に抑える
- コンテンツを輝かせる設計
- UIは「黒子」に徹する

**チェックリスト**:
- [ ] コンテンツが主役になっているか？
- [ ] UI要素が過度に目立っていないか？
- [ ] 装飾がコンテンツより目立っていないか？
- [ ] 半透明やブラー効果を適切に使用しているか？

---

### 3. Depth（深度）

**定義**: 視覚的なレイヤーとリアルな動きで階層を伝え、理解を促進

**実践ポイント**:
- 視差効果やブラー処理の活用
- 要素間の関係性を直感的に把握できる
- レイヤーによる情報階層の表現
- アニメーションで空間的な関係を示す

**チェックリスト**:
- [ ] レイヤー構造は理解しやすいか？
- [ ] モーダルやシートの階層が明確か？
- [ ] アニメーションが空間的関係を表現しているか？
- [ ] ブラー効果が適切に使用されているか？

---

## 6つの設計原則（1984年Macintosh以来）

### 1. Aesthetic Integrity（美的一貫性）

**定義**: アプリの外観がその機能といかに統合されているかを測る尺度

**適用指針**:

| アプリタイプ | 推奨スタイル |
|------------|-------------|
| 生産性アプリ | 装飾を控えめに |
| ゲーム | 発見を促す魅力的な外観 |
| ユーティリティ | 機能優先のシンプルさ |
| クリエイティブ | ツールとキャンバスのバランス |

**チェックリスト**:
- [ ] 見た目と機能が調和しているか？
- [ ] アプリの目的にふさわしい外観か？
- [ ] 装飾が機能を阻害していないか？

---

### 2. Consistency（一貫性）

**定義**: ユーザーがあるアプリから別のアプリへ知識を転用できる

**実践要素**:
- システム提供のインターフェース要素を使用
- よく知られたアイコン
  - 設定 → 歯車
  - 検索 → 虫眼鏡
  - 共有 → 四角から矢印
  - 戻る → 左向き矢印
- 標準的なテキストスタイル
- プラットフォーム標準のジェスチャー

**チェックリスト**:
- [ ] システム標準のコンポーネントを使用しているか？
- [ ] アイコンは広く認知されているものか？
- [ ] 同じアクションに同じ操作方法を使用しているか？
- [ ] プラットフォームの慣習に従っているか？

---

### 3. Direct Manipulation（直接操作）

**定義**: 画面上のオブジェクトを直接操作することで、タスクにより深く関与

**標準的なジェスチャー**:

| ジェスチャー | アクション |
|------------|----------|
| タップ | 選択、アクティベート |
| スワイプ | スクロール、ナビゲーション |
| ピンチイン/アウト | ズームイン/アウト |
| 長押し | コンテキストメニュー |
| ドラッグ | 移動、並べ替え |

**チェックリスト**:
- [ ] オブジェクトを直接操作できるか？
- [ ] アクションの結果が即座に表示されるか？
- [ ] ジェスチャーが直感的か？
- [ ] 中間的なダイアログを最小化しているか？

---

### 4. Feedback（フィードバック）

**定義**: ユーザーのアクションを認識し、処理が進行中であることを保証

**フィードバックの種類**:

| 種類 | 用途 | 例 |
|-----|------|-----|
| 視覚的ハイライト | タップの認識 | ボタンの色変化 |
| プログレスバー | 長時間処理 | ダウンロード進捗 |
| アニメーション | 状態変化 | 要素の移動 |
| 色の変化 | 状態表示 | 選択状態 |
| 触覚フィードバック | 確認 | Haptics |

**チェックリスト**:
- [ ] タップに即座に反応しているか？
- [ ] 処理中であることが明確か？
- [ ] 成功/失敗が視覚的に分かるか？
- [ ] フィードバックは適切な強さか？

---

### 5. Metaphors（メタファー）

**定義**: 実世界や既存のデジタル体験から連想して、より早く学習

**代表的なメタファー**:

| メタファー | 実世界の対応 | アプリ例 |
|----------|------------|---------|
| フォルダ | 書類整理 | ファイル管理 |
| 封筒 | 手紙 | メール |
| ゴミ箱 | くずかご | 削除 |
| ページ | 本のページ | 電子書籍 |
| スライダー | ボリュームつまみ | 設定調整 |

**チェックリスト**:
- [ ] メタファーは広く理解されるものか？
- [ ] 実世界の操作と矛盾していないか？
- [ ] メタファーが機能を正しく表現しているか？
- [ ] 過度に複雑なメタファーを避けているか？

---

### 6. User Control（ユーザーコントロール）

**定義**: アプリではなく人がコントロールする

**ユーザーコントロールの原則**:

**すべきこと**:
- 行動を提案する
- 危険を警告する
- 選択肢を提示する
- 確認を求める（破壊的アクション時）

**してはならないこと**:
- 意思決定を奪う
- 勝手に動作を変更する
- 元に戻せないアクションを警告なしに実行
- ユーザーの意図を無視する

**チェックリスト**:
- [ ] ユーザーが常にコントロールしているか？
- [ ] 重要な決定はユーザーに委ねているか？
- [ ] 取り消し（Undo）が可能か？
- [ ] 強制的な動作を避けているか？

---

## 原則の適用優先順位

1. **User Control** - ユーザーの自律性を最優先
2. **Clarity** - 理解できなければ使えない
3. **Feedback** - 応答がなければ不安になる
4. **Consistency** - 学習コストを下げる
5. **Direct Manipulation** - 効率と没入感を高める
6. **Aesthetic Integrity** - 全体の完成度を上げる
7. **Metaphors** - 学習を補助する
