## 1. 3つのテーマ

### Clarity（明瞭性）

- [ ] テキストはすべてのサイズで読みやすい
- [ ] アイコンの意味が明確で理解しやすい
- [ ] 装飾は控えめで適切
- [ ] 視覚的階層が明確（重要な要素が目立つ）
- [ ] 曖昧さや混乱を招く要素がない

### Deference（謙譲性）

- [ ] コンテンツが主役になっている
- [ ] UI要素は控えめで、コンテンツを邪魔しない
- [ ] ボーダーや影は最小限
- [ ] 半透明やブラー効果が適切に使用されている

### Depth（深度）

- [ ] レイヤー構造が直感的に理解できる
- [ ] モーダルやシートの階層関係が明確
- [ ] アニメーションが空間的な関係を表現している
- [ ] 視差効果やブラーが適切に使用されている

## 2. 6つの設計原則

### Aesthetic Integrity（美的一貫性）

- [ ] アプリの外観がその機能と調和している
- [ ] アプリタイプ（生産性/ゲーム/ユーティリティ）にふさわしいスタイル
- [ ] 装飾が機能を阻害していない

### Consistency（一貫性）

- [ ] システム標準のコンポーネントを使用している
- [ ] よく知られたアイコン（歯車、虫眼鏡等）を使用
- [ ] プラットフォームの慣習に従っている
- [ ] アプリ内で一貫したスタイルを維持

### Direct Manipulation（直接操作）

- [ ] オブジェクトを直接操作できる
- [ ] アクションの結果が即座に表示される
- [ ] ジェスチャーが直感的
- [ ] 中間的なダイアログを最小化している

### Feedback（フィードバック）

- [ ] タップに即座に反応する
- [ ] 処理中であることが明確に表示される
- [ ] 成功/失敗が視覚的に分かる
- [ ] フィードバックの強さが適切

### Metaphors（メタファー）

- [ ] 使用するメタファーが広く理解されている
- [ ] 実世界の操作と矛盾していない
- [ ] メタファーが機能を正しく表現している

### User Control（ユーザーコントロール）

- [ ] ユーザーが常にコントロールしている
- [ ] 重要な決定はユーザーに委ねている
- [ ] 取り消し（Undo）が可能
- [ ] 強制的な動作を避けている

## 3. UIコンポーネント

### タッチターゲット

- [ ] **最小44×44pt**（visionOSは60×60pt）
- [ ] タップしやすい間隔が確保されている
- [ ] 視覚的に小さくてもタッチ領域は十分

### ナビゲーション

**iOS**:
- [ ] タブバーが画面下部に配置（3〜5タブ）
- [ ] 具体的なラベルを使用（「Home」は避ける）
- [ ] 戻るボタンが左上に配置

**iPad**:
- [ ] サイドバーナビゲーションを検討
- [ ] Split View対応を検討

### ボタン

- [ ] タイトルに動詞を使用
- [ ] プライマリアクションは青色
- [ ] 破壊的アクションは赤色
- [ ] タイトルは短く簡潔

### モーダル/シート

- [ ] 使用は最小限
- [ ] 完了ボタンとキャンセルボタンがある
- [ ] タスクがシンプルで焦点が絞られている
- [ ] 背景が適切に暗くなる

### アラート

- [ ] 重要な情報にのみ使用
- [ ] タイトルは短い（1行以内）
- [ ] ボタンは1〜2個
- [ ] 乱用していない

### アクションシート

- [ ] キャンセルボタンがある
- [ ] 破壊的アクションは赤色で上部に配置
- [ ] ボタン数は最大4個

## 4. コンポーネント状態

### 8つの状態定義

- [ ] **Default**: 標準外観が定義されている
- [ ] **Hover**: ポインター接触時の変化（macOS/iPadOS）
- [ ] **Pressed**: タップ/クリック中の視覚変化（暗く/縮小）
- [ ] **Focused**: キーボード/VoiceOverフォーカス時の青枠
- [ ] **Selected**: 選択状態のアクセント色表示
- [ ] **Disabled**: 無効状態の50%透明度
- [ ] **Loading**: 読み込み中のスピナー/シマー
- [ ] **Error**: エラー状態の赤色/アイコン表示

### 状態遷移

- [ ] アニメーション遷移が設定されている（100-200ms）
- [ ] 触覚フィードバックが適切に設定されている
- [ ] 状態間の遷移が滑らか

## 5. ビジュアルデザイン

### 角丸（Squircle）

- [ ] Apple Squircle（連続曲線）を使用
- [ ] 標準値を使用（8, 10, 12, 14, 16, 20, 24pt）
- [ ] コンポーネントサイズに適した角丸

### シャドウ

- [ ] 6段階のシャドウレベルを適切に使用
  - [ ] Subtle（3-4px blur）
  - [ ] Light（6-8px blur）
  - [ ] Medium（10-16px blur）
  - [ ] Strong（16-24px blur）
  - [ ] Heavy（24-32px blur）
  - [ ] Maximum（32-48px blur）
- [ ] ダークモードでシャドウが適切に調整されている

### スペーシング

- [ ] 4pt/8ptグリッドシステムを使用
- [ ] 一貫したパディング（8, 12, 16, 20, 24pt）
- [ ] 要素間の適切な間隔

### アニメーション

- [ ] 適切なduration（100-500ms）
- [ ] ease-out/ease-in-out easingを使用（linear回避）
- [ ] Reduce Motion対応
- [ ] モーダル/シートのトランジション（300-400ms）

## 8. アクセシビリティ

### VoiceOver

- [ ] すべてのUI要素がVoiceOverでナビゲート可能
- [ ] 適切なaccessibilityLabelが設定されている
- [ ] accessibilityHintで追加コンテキストを提供
- [ ] 「ボタン」という語を手動で追加していない
- [ ] accessibilityTraitsが正しく設定されている

### キーボード

- [ ] フォーカス順序が論理的
- [ ] キーボードショートカットがある（macOS）
- [ ] フォーカスインジケーターが明確（3px青枠）

### Reduce Motion

- [ ] Reduce Motion設定時にアニメーションが簡素化される
- [ ] 回転やスケーリングがフェードに置き換わる
- [ ] 情報がモーションのみに依存しない

### 高コントラスト

- [ ] 高コントラストモードで視認性が向上
- [ ] ボーダーやセパレータが明確になる
