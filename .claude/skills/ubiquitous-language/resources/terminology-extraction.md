# 用語抽出の手法

## 概要

ユビキタス言語を確立するための最初のステップは、ドメインで使用される用語を
網羅的に収集することです。このドキュメントでは、効果的な用語抽出の手法を解説します。

## 用語抽出の情報源

### 1. ドメインエキスパートとの対話

**最も重要な情報源**

**インタビュー手法**:

- オープンクエスチョンで業務プロセスを聞く
- 具体的なシナリオを描写してもらう
- 「それを何と呼びますか？」と頻繁に確認
- 専門用語が出たらメモし、定義を確認

**ポイント**:

- 技術者の言葉に言い換えない
- 相手の言葉をそのまま記録
- 同じ概念に複数の呼び方がないか確認

### 2. 既存ドキュメント

**対象**:

- 業務マニュアル
- 仕様書
- 契約書
- ヘルプドキュメント

**抽出方法**:

1. 名詞を抽出（概念、エンティティ候補）
2. 動詞を抽出（操作、プロセス候補）
3. 状態を表す形容詞を抽出
4. 専門用語や固有名詞を特定

### 3. 既存システム

**対象**:

- データベーステーブル名
- 画面ラベル
- エラーメッセージ
- レポートのヘッダー

**注意点**:

- 技術的な名前（id, created_at等）は除外
- ビジネス用語のみを抽出
- 既存システムの用語が正しいとは限らない

### 4. 業務フロー図

**対象**:

- ワークフロー図
- シーケンス図
- 状態遷移図

**抽出要素**:

- プロセス名
- 状態名
- アクター名
- イベント名

## 用語の分類

### 名詞（エンティティ/値オブジェクト候補）

| 分類         | 説明               | 例                   |
| ------------ | ------------------ | -------------------- |
| 核心ドメイン | ビジネスの中心概念 | 注文、顧客、商品     |
| 支援ドメイン | 核心を支える概念   | 住所、金額、日付範囲 |
| 汎用         | 一般的な概念       | ユーザー、通知、設定 |

### 動詞（操作/プロセス候補）

| 分類           | 説明             | 例                           |
| -------------- | ---------------- | ---------------------------- |
| ライフサイクル | 作成・変更・削除 | 登録する、更新する、取り消す |
| 状態遷移       | 状態の変化       | 承認する、却下する、完了する |
| クエリ         | 情報の取得       | 検索する、一覧する、参照する |

### 状態（ステータス候補）

| 分類     | 説明         | 例                         |
| -------- | ------------ | -------------------------- |
| 初期状態 | 開始時の状態 | 下書き、未処理、保留中     |
| 中間状態 | 処理中の状態 | 処理中、承認待ち、配送中   |
| 終端状態 | 完了時の状態 | 完了、キャンセル、期限切れ |

## 用語抽出のプロセス

### Step 1: 収集

```
1. 情報源をリストアップ
2. 各情報源から用語を抽出
3. 重複を含むリストを作成
4. 文脈も一緒に記録
```

### Step 2: 整理

```
1. 重複を統合
2. 同義語を特定
3. 曖昧な用語をマーク
4. カテゴリ分類
```

### Step 3: 検証

```
1. ドメインエキスパートにレビュー依頼
2. 定義の確認
3. 不足している用語の追加
4. 不要な用語の削除
```

## 抽出時の注意点

### 避けるべきこと

1. **技術用語への変換**
   - ❌ 「ワークフロー」→「プロセスオブジェクト」
   - ✅ ドメインエキスパートの言葉をそのまま使用

2. **一般化しすぎ**
   - ❌ 「注文」「予約」「申込」→「リクエスト」
   - ✅ 各概念を区別して保持

3. **文脈の無視**
   - ❌ 用語だけをリスト化
   - ✅ どの文脈で使われるかも記録

### 心がけること

1. **網羅性**
   - 主要な概念だけでなく細かい概念も
   - 状態、イベント、関係性も含める

2. **正確性**
   - ドメインエキスパートの言葉を正確に
   - 解釈を加えない

3. **追跡可能性**
   - 情報源を記録
   - 後から検証可能に
