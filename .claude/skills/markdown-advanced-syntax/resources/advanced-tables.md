# 高度なテーブル構造

## 概要

Markdownテーブルを効果的に使用し、データ構造、API仕様、比較情報を明確に表現する手法です。

## 基本構文

```markdown
| 左揃え | 中央揃え | 右揃え |
|:-------|:--------:|-------:|
| データ | データ   | データ |
```

## 整列の使い分け

| 整列 | 用途 | 例 |
|:-----|:-----|:---|
| 左揃え (`:---`) | テキスト、説明 | 名前、概要 |
| 中央揃え (`:---:`) | ステータス、ラベル | Yes/No、状態 |
| 右揃え (`---:`) | 数値、金額 | 価格、件数 |

## API仕様テーブルパターン

### エンドポイント一覧

| メソッド | パス | 概要 | 認証 |
|:---------|:-----|:-----|:----:|
| GET | `/api/users` | ユーザー一覧取得 | ○ |
| POST | `/api/users` | ユーザー作成 | ○ |
| GET | `/api/users/{id}` | ユーザー詳細取得 | ○ |

### パラメータ定義

| パラメータ | 型 | 必須 | 説明 | デフォルト |
|:-----------|:---|:----:|:-----|:-----------|
| `page` | integer | - | ページ番号 | 1 |
| `limit` | integer | - | 取得件数 | 20 |
| `sort` | string | - | ソート順 | `created_at` |

### レスポンスフィールド

| フィールド | 型 | 説明 |
|:-----------|:---|:-----|
| `id` | string (UUID) | ユーザーID |
| `name` | string | ユーザー名 |
| `email` | string | メールアドレス |
| `created_at` | string (ISO 8601) | 作成日時 |

## データモデルテーブル

### エンティティ定義

| フィールド | 型 | 制約 | 説明 |
|:-----------|:---|:-----|:-----|
| `id` | UUID | PK, NOT NULL | 主キー |
| `name` | VARCHAR(100) | NOT NULL | 名前 |
| `status` | ENUM | DEFAULT 'active' | ステータス |

## ベストプラクティス

### すべきこと
- 列数は10以下に抑える
- ヘッダーを明確に
- 空セルには `-` を入れる

### 避けるべきこと
- 長すぎるセル内容
- 複雑なマークアップの埋め込み
- 整列の不統一
