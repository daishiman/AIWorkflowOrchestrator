# ナビゲーションパターンガイド

## 概要

このガイドは、ドキュメントサイトにおける効果的な
ナビゲーション設計パターンを詳細に説明します。

## ナビゲーションの目的

1. **現在地の明示**: ユーザーが今どこにいるかを示す
2. **移動手段の提供**: 目的の場所へ移動する方法を提供
3. **全体構造の理解**: サイト全体の構造を把握させる
4. **発見の促進**: 関連コンテンツを発見させる

## グローバルナビゲーション

### 設計原則

1. **一貫性**: すべてのページで同じ位置に配置
2. **簡潔さ**: 主要セクションのみ表示（5-7項目）
3. **予測可能性**: 標準的なラベルを使用

### 推奨構造

```
[ホーム] [ドキュメント] [チュートリアル] [API] [コミュニティ] [検索]
```

## パンくずリスト

### 基本形式

```
ホーム > セクション > サブセクション > 現在のページ
```

### 実装例

```markdown
[ホーム](/) > [ドキュメント](/docs) > [ガイド](/docs/guides) > **認証設定**
```

### パンくずのベストプラクティス

| Do | Don't |
|----|-------|
| すべての階層を表示 | 途中を省略する |
| クリック可能なリンク | 現在ページもリンク |
| 簡潔なラベル | 長いページタイトル |
