# 情報アーキテクチャ

## 概要

文書全体の構造設計です。読者が必要な情報を効率的に見つけられるよう、コンテンツを組織化します。

## 設計原則

### 1. ユーザー中心

読者の目的とタスクに基づいて構造を設計します。

| 読者の質問         | 提供する構造    |
| :----------------- | :-------------- |
| 「始め方は？」     | Getting Started |
| 「どうやるの？」   | How-to Guides   |
| 「詳細仕様は？」   | Reference       |
| 「なぜこうなる？」 | Concepts        |

### 2. 段階的開示

情報を段階的に提示し、読者を圧倒しません。

```
レベル1: 概要（全体像）
  └─ レベル2: カテゴリ（機能グループ）
      └─ レベル3: トピック（個別機能）
          └─ レベル4: 詳細（オプション・例外）
```

### 3. 論理的グループ化

関連する情報をまとめ、無関係な情報を分離します。

```
✅ 論理的グループ
認証
├── 概念: 認証とは
├── 設定: OAuth設定
├── 設定: API Key設定
└── トラブルシューティング

❌ 機能別の羅列
├── OAuth
├── データベース
├── API Key
├── キャッシュ
```

## 構造パターン

### Diátaxis フレームワーク

| 象限              | 目的     | 特徴             |
| :---------------- | :------- | :--------------- |
| **Tutorials**     | 学習     | 手を動かして学ぶ |
| **How-to Guides** | 問題解決 | 特定タスクの手順 |
| **Reference**     | 情報     | 技術仕様、API    |
| **Explanation**   | 理解     | 背景、概念、理由 |

## ナビゲーション設計

### グローバルナビゲーション

常に表示される主要なナビゲーション。

```
[ホーム] [ガイド] [API] [例] [FAQ]
```

### ローカルナビゲーション

現在のセクション内のナビゲーション。

```
ガイド
├── はじめに ← 現在地
├── インストール
├── 設定
└── 使い方
```

### コンテキストナビゲーション

関連コンテンツへのリンク。

```markdown
## 関連情報

- [次のステップ: 高度な設定](./advanced-config.md)
- [参考: API仕様](../reference/api.md)
- [トラブルシューティング](../troubleshooting/common-issues.md)
```

### パンくずリスト

現在位置の階層表示。

```
ホーム > ガイド > インストール > Linux
```

## 命名と整理

### ファイル命名規則

| パターン               | 例                   | 用途         |
| :--------------------- | :------------------- | :----------- |
| `{動詞}-{対象}.md`     | `configure-ssl.md`   | タスク       |
| `{対象}-overview.md`   | `auth-overview.md`   | 概要         |
| `{対象}-reference.md`  | `api-reference.md`   | リファレンス |
| `{番号}-{タイトル}.md` | `01-introduction.md` | 順序付き     |

### ディレクトリ命名

```
# 小文字、ハイフン区切り
✅ how-to-guides/
✅ api-reference/
❌ How_To_Guides/
❌ APIReference/
```

### インデックスファイル

各ディレクトリにインデックスを配置。

```
guides/
├── index.md            # ガイド一覧と概要
├── getting-started.md
├── configuration.md
└── deployment.md
```

## 検索性の最適化

### メタデータ

```yaml
---
title: OAuth 2.0の設定

keywords: [oauth, 認証, 設定, セキュリティ]
category: how-to
audience: developer
---
```

### 相互リンク

```markdown
- エラーからトラブルシューティングへ
```
